# 메이크로컬 MVP 제품기획서 v0.1

## 1. 제품 개요

메이크로컬은 시제품 제작이 필요한 연구소, 하드웨어 스타트업, 예비창업자와 지역 중소 제조업체를 연결하는 AI 기반 제조 매칭 서비스다.

초기 MVP는 완전 자동 견적 플랫폼이 아니라, 사용자가 도면 또는 제작 요구사항을 입력하면 AI가 제작의뢰서 초안을 생성하고, 예상 제작방법, 예상 견적 범위, 예상 납기 일정을 제안하며, 지역 제조업체 DB를 기반으로 적합한 업체 후보를 추천하는 운영형 파일럿 서비스로 시작한다.

장기적으로는 실제 견적 데이터, 제작 사례, 업체 응답률, 공정별 적합도 데이터를 축적하여 자동 견적과 제조업체 추천 정확도를 높이는 제조 매칭 플랫폼으로 확장한다.

## 2. 문제 정의

시제품 제작자는 도면이나 아이디어가 있어도 어떤 제조업체가 해당 부품을 제작할 수 있는지 찾기 어렵다. 특히 연구소, 스타트업, 예비창업자는 기존 거래처나 제조 네트워크가 부족한 경우가 많아 업체 탐색, 도면 전달, 견적 요청, 제작방법 협의에 많은 시간을 사용한다.

제조업체 입장에서도 문제는 있다. 제작 의뢰가 들어와도 재질, 수량, 공차, 후처리, 납기, 조립 여부, 사용 목적 등 견적에 필요한 정보가 누락되어 있으면 반복적으로 확인해야 한다. 이 과정은 견적 지연, 커뮤니케이션 비용 증가, 제작 오류 가능성으로 이어진다.

반대로 지역 중소 제조업체는 선반, 밀링, MCT, 판금, 용접, 표면처리 등 다양한 제조 역량을 보유하고 있음에도 온라인 수주 채널이 부족해 소수 고객사에 의존하는 경우가 많다. 이로 인해 수주량이 불안정하고, 기술력이 있어도 신규 고객을 만나기 어렵다.

메이크로컬은 이 양쪽의 문제를 연결해서 해결하고자 한다.

## 3. 창업자 적합성

창업자는 10년 이상 기계·자동차 분야 연구소에서 설계 및 개발 업무를 수행했다. 중견기업부터 스타트업까지 다양한 조직에서 제품 설계, 시제품 제작, 제작업체 소싱, 업체 개발, 도면 전달, 견적 협의, 제작 일정 조율을 경험했다.

또한 가족을 통해 지역 기계가공업의 현실을 가까이서 보았다. 아버지는 오랜 기간 대구 3공단에서 선반·밀링 등 기계가공 분야에 종사했으며, 현장에는 숙련된 기술자와 가공 역량이 있음에도 안정적인 수주 채널이 부족해 어려움을 겪는 경우가 많았다.

따라서 창업자는 시제품 제작 수요자와 제조업체 양쪽의 문제를 모두 이해하고 있으며, 이 경험을 바탕으로 서비스를 설계한다.

## 4. 핵심 고객

### 4.1 1차 고객: 시제품 제작 수요자

- 대학 연구소
- 하드웨어 스타트업
- 예비창업자
- 제품 개발자
- 창업보육센터 입주기업
- 메이커스페이스 이용자

이들은 도면이나 아이디어는 있지만 적합한 제작업체를 찾기 어렵고, 제작업체가 검토하기 쉬운 형태로 요구사항을 정리하는 데 어려움을 겪는다.

### 4.2 2차 고객/공급자: 지역 중소 제조업체

- 선반 가공 업체
- 밀링/MCT/CNC 가공 업체
- 판금 업체
- 용접 업체
- 표면처리 업체
- 지그 제작 업체
- 3D프린팅 및 시제품 제작 업체

이들은 제조 역량은 있지만 온라인 수주 채널과 신규 고객 접점이 부족한 경우가 많다.

### 4.3 초기 실증 지역

초기에는 창업자가 접근하기 쉽고 문제를 잘 이해하고 있는 대구권 제조업체, 특히 대구 3공단 및 대구권 가공업체를 중심으로 파일럿을 진행한다. 이후 성서산단, 달성산단, 경북권, 전국 산업단지로 확장 가능성을 검토한다.

## 5. 핵심 가치 제안

### 5.1 시제품 제작자에게 제공하는 가치

- 어떤 제조업체에 문의해야 할지 찾는 시간을 줄인다.
- 도면과 요구사항을 제작업체가 이해하기 쉬운 의뢰서로 정리한다.
- 예상 제작방법, 예상 견적 범위, 예상 납기 일정을 미리 확인할 수 있다.
- 누락 정보와 견적 상승 요인을 사전에 확인할 수 있다.
- 적합한 지역 제조업체 후보를 추천받을 수 있다.

### 5.2 제조업체에게 제공하는 가치

- 표준화된 제작의뢰서를 통해 견적 검토 시간을 줄인다.
- 재질, 수량, 공차, 후처리, 납기 등 핵심 정보가 정리된 의뢰를 받을 수 있다.
- 신규 고객과 시제품 제작 수요를 만날 수 있다.
- 온라인 수주 채널을 확보할 수 있다.

## 7. 사용자 흐름

### 7.1 의뢰자 흐름

1. 사용자가 랜딩페이지에 접속한다.
2. “제작 의뢰 시작하기” 버튼을 클릭한다.
3. 파일 업로드 페이지로 이동한다.
4. PDF, STEP, STL, JPG, PNG 파일을 업로드한다.
5. 파일이 없는 경우 “도면 없이 의뢰서 작성하기”를 선택할 수 있다.
6. 제작의뢰서 자동 생성 페이지로 이동한다.
7. AI가 파일 정보와 입력 정보를 바탕으로 제작의뢰서 초안을 생성한다.
8. AI가 알 수 없는 항목은 빈칸 또는 미입력 상태로 남긴다.
9. 사용자는 소재, 수량, 공차, 후처리, 희망 납기 등 필요한 항목을 수정하거나 입력한다.
10. AI가 예상 제작방법, 예상 견적 범위, 예상 납기 일정, 견적 상승 요인, 업체 확인 필요사항을 제안한다.
11. 사용자는 제작의뢰서를 확정하고 다음 단계로 이동한다.
12. 시스템은 제조업체 DB를 기준으로 적합한 업체 후보를 추천한다.
13. 사용자는 추천 업체 중 하나를 선택하거나 “운영자에게 맡기기”를 선택한다.
14. 최종 요약 페이지에서 입력 정보를 확인한다.
15. 이름, 회사명/소속, 이메일, 휴대폰 번호, 연락 선호 방식 등 연락처를 입력한다.
16. “제작 문의 제출하기” 버튼을 클릭한다.
17. 제출 완료 페이지에서 접수 완료 안내를 확인한다.

### 7.2 관리자 흐름

1. 관리자가 관리자 페이지에 접속한다.
2. 접수된 제작 의뢰 목록을 확인한다.
3. 특정 의뢰의 상세 페이지로 이동한다.
4. 첨부 파일, AI 생성 제작의뢰서, 사용자가 수정한 최종 의뢰서를 확인한다.
5. AI 추천 업체 목록과 사용자가 선택한 업체를 확인한다.
6. 필요한 경우 제조업체 DB에서 추가 업체를 검색한다.
7. 추천 업체를 확정하거나 운영자 메모를 남긴다.
8. 의뢰 상태를 변경한다.
9. 이후 실제 연락 및 견적 연결은 운영자가 수동으로 진행한다.

## 9. 데이터 저장 원칙

MVP 단계부터 데이터는 향후 자동 견적과 업체 추천 고도화에 활용할 수 있도록 구조적으로 저장한다.

저장해야 할 주요 데이터는 다음과 같다.

- 첨부 파일 정보
- 원본 입력값
- AI 생성 제작의뢰서
- 사용자가 수정한 최종 제작의뢰서
- AI 추천 제작방법
- 예상 견적 범위
- 예상 납기 일정
- 누락 정보 체크리스트
- 추천 업체 목록
- 사용자가 선택한 업체
- 최종 제출 연락처
- 운영자 상태값
- 운영자 메모

## 10. 상태값 정의

제작 의뢰 상태는 다음과 같이 관리한다.

- `submitted` : 접수 완료
- `reviewing` : 운영자 검토 중
- `needs_info` : 정보 보완 필요
- `supplier_recommended` : 업체 추천 완료
- `quote_requested` : 견적 요청 중
- `quote_received` : 견적 회신 완료
- `in_production` : 제작 진행 중
- `completed` : 완료
- `on_hold` : 보류
- `cancelled` : 취소

## 11. 초기 기술 스택

- Frontend/App: Next.js, TypeScript, Tailwind CSS
- Backend/DB/Auth/Storage: Supabase
- AI: OpenAI API 또는 교체 가능한 AI API 구조
- Deploy: Vercel
- Version Control: GitHub

초기에는 회원가입과 로그인을 보류하므로 Supabase Auth는 바로 사용하지 않는다. 다만 향후 확장을 고려해 사용자/관리자 구조가 추가될 수 있도록 설계한다.

## 12. 파일 업로드 정책

초기 허용 파일 형식은 다음과 같다.

- PDF
- STEP
- STL
- JPG
- PNG

ZIP 파일은 초기 MVP에서 제외한다.

파일 없이도 다음 단계로 이동할 수 있어야 한다. 이는 도면이 아직 없는 초기 창업자나 연구자가 텍스트 설명만으로도 의뢰를 시작할 수 있게 하기 위함이다.

## 13. AI 응답 원칙

AI는 확정값을 제공하는 것이 아니라, 제작의뢰서 초안과 예상 범위를 제공한다.

AI가 확실히 알 수 없는 정보는 추정하지 않고 빈칸, 미입력, 확인 필요 등으로 표시한다.

예상 견적은 다음과 같이 범위로 표현한다.

- 예: 100만 원 ~ 200만 원 수준
- 실제 견적은 제조업체 검토 후 확정됩니다.

예상 납기도 다음과 같이 범위로 표현한다.

- 예: 영업일 기준 7~14일
- 소재 수급, 후처리, 도면 보완 여부에 따라 변동될 수 있습니다.

AI 결과는 반드시 사용자가 수정할 수 있어야 한다.

## 14. 초기 수익모델 가설

MVP 검증 이후 다음 수익모델을 검토한다.

1. 제작의뢰서/견적 검토 리포트 비용
2. 제작 연결 성사 수수료
3. 제조업체 등록 및 홍보 페이지 비용
4. 창업보육센터·대학·기관 대상 시제품 제작 상담 프로그램
5. 프리미엄 업체 매칭 서비스

초기에는 수익화보다 실제 의뢰와 업체 반응 검증을 우선한다.

## 17. 현재 결정된 사항

- 1차 MVP에서는 회원가입 기능을 보류한다.
- 1차 MVP에서는 로그인 기능을 보류한다.
- 제조업체 회원가입도 보류한다.
- 파일 업로드 형식은 PDF, STEP, STL, JPG, PNG만 허용한다.
- ZIP 파일은 제외한다.
- AI 기능은 제작방법 추천, 예상 견적 범위, 예상 납기 일정, 제작의뢰서 초안 생성으로 시작한다.
- 예상 견적은 확정 금액이 아니라 범위로 표시한다.
- 제조업체 DB는 단순한 구조로 시작하고 필요 시 확장한다.
- 초기 서비스는 정식 서비스가 아니라 파일럿 제작 의뢰 접수 형식으로 운영한다.
